© All in Books 2026  •  Confidential  Page 2 1. Executive Summary & Vision  
All in Books is a free, offline -first Progressive Web App (PWA) for personal and small -business accounting 
in India. It runs on Android Chrome (installable PWA) and any desktop web browser — one codebase, no 
backend server, no app store. Currency is fixed to INR (₹). The user always owns their data, stored on -
device and backed up to their own Google Drive.  
 
"All in Books should feel like the best ledger book you have ever used — except it lives on 
your Android phone, never loses a page, and hands you a professional report whenever you 
need one."

© All in Books 2026  •  Confidential  Page 6 4. User Stories & Acceptance Criteria  
4.1  Journal Entries  
US-001  As a shop owner, I want to quickly record a sale transaction, so that I have an accurate 
searchable record of all income.  
Acceptance Criteria:  
1. Entry form opens within 1 tap via FAB from any screen  
2. Required fields: type and amount only — all others optional  
3. Entry type defaults to last -used; date defaults to today (editable)  
4. Transaction saved to IndexedDB immediately on tapping Save — no server call  
5. Green toast 'Saved ✓' appears for 2 seconds and auto -dismisses  
 
US-002  As a GST -registered business owner, I want to record the GST component separately from 
the base amount, so that I can calculate my net GST liability at any time.  
Acceptance Criteria:  
6. GST toggle defaults Off; toggling On shows rate chips: 0%, 5%, 12%, 18%, 28%  
7. User enters total (GST -inclusive); base and GST auto -calculated live via calcFromTotal()  
8. Both baseAmount and gstAmount stored as separate numeric fields in IndexedDB  
9. If GST off, gstRate and gstAmount stored as null — not zero  
10. GST breakdown visible in entry detail, ledger export, and GST Summary report  
 
US-003  As a user, I want to attach a photo of a bill to any entry, so that I have a permanent digital 
copy without keeping paper.  
Acceptance Criteria:  
11. Camera (Android) or gallery file selection available on every entry form  
12. Files over 50 MB blocked with toast error: 'File too large. Max size is 50 MB.'  
13. Valid attachments compressed to under 400 KB before IndexedDB storage  
14. Thumbnail shown in entry list; tapping opens full -res lightbox via Modal  
 
4.2  Historical Data Import  
US-004  As a user migrating from paper or Excel, I want to upload a structured Excel file to import all 
historical data, so that I do not lose years of records when switching to All in Books.  
Acceptance Criteria:  
15. Import accessible from Settings → Data & Backup → Import Historical Data  
16. App provides downloadable import template (.xlsx) — always available offline  
17. Template has 5 sheets: Transactions, Party_Balances, Parties, GST_History, README  
18. Pre-import review screen: green (valid), red (error) rows with per -cell error messages  
19. User may proceed with valid rows only, or cancel entirely  
20. Post-import summary: Imported N, Skipped (duplicates) N, Errors N  
21. 'Undo Import' button available for 24 hours after import completes

34. Offline dot visible in header when navigator.onLine is false  
35. Google Drive sync fires automatically in background when connectivity restores  
36. No loading spinners or degraded UI due to offline status

39. File named: AllInBooks_Export_[MMM -YYYY].xlsx

Sheet 1: Transactions  
Columns: date (YYYY -MM-DD, required), type (sale|purchase|expense|receipt|payment, required), 
amount (positive number, required), baseAmount (optional), gstRate (0/5/12/18/28/blank, optional), 
gstAmount (optional), partyName (optional), category (optional) , note (max 500 chars), invoiceNumber 
(optional).  
Sheet 2: Party_Balances  
Columns: partyName (required), balanceDate (YYYY -MM-DD, required), balanceAmount (positive, 
required), balanceType (toReceive|toPay, required), note (optional).  
Sheet 3: Parties  
Columns: name (required), phone (optional), address (optional), gstin (optional).  
Sheet 4: GST_History  
Columns: period (e.g. Q3 FY2025, required), outputGST (number, required), inputGST (number, 
required), gstRate (optional), note (optional).  
Sheet 5: README  
Non-data sheet. Plain English step -by-step guide, field instructions, example data. Row 1=headers, Row 
2=format hints, Row 3=example in all data sheets.

7.1  Data Flow — Local -First  
User Action → Dexie.js (IndexedDB, instant, offline) → UI re -renders → [scheduled] SheetJS Export → 
Google Drive Backup. No user -facing operation ever waits on a network response.  
 
7.2  PWA Caching Strategy  
• App assets (JS / CSS / fonts / icons): Cache -First — always served from service worker cache  
• Google Drive API calls: Network -First with 10 -second timeout — falls back gracefully  
• Initial JS bundle target: < 500 KB transferred (Vite bundle analyser)  
• Android Chrome: PWA install prompt captured via beforeinstallprompt event
